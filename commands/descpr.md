# 手順

0. ユーザから引数が渡されていない場合は以下を聞く
    ```
    PRの番号を教えてください。
    ```

1. 該当のPRの変更ファイルとその内容を確認する

2. 変更内容を元にPRの概要を更新する
    * 概要は以下のフォーマットに従って日本語で記述すること
        ```markdown
        # 概要
        <!-- PRの概要を簡潔に記述する -->
        
        # やったこと
        <!-- PRの変更内容を記述する -->
        
        # やらなかったこと
        <!-- このPRではやらなかったことを記述する -->
        <!-- なぜやらなかったかの理由まで記述すること -->
        
        # レビューの観点
        <!-- このPRをレビューする際に見てほしい観点を記述する -->
        
        # 備考
        <!-- 公式ドキュメントなど参考情報がある場合は記述する -->
        ```

# 注意点

* 手順1, 2は`gh api`コマンドを用いて実行すること
* `gh pr`コマンドは使用しないこと
