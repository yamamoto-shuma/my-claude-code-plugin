# バージョン更新の基準

バージョン番号は {MAJOR}.{MINOR}.{PATCH} という3つの数字で構成されます（例：1.0.0）

1. PATCH（パッチ）を上げる：1.0.0 → 1.0.1
    * 内容： 後方互換性があるバグ修正
    * タイミング：
        * タイポ（誤字）の修正
        * 内部的な処理の改善（リファクタリング）
        * 不具合の修正（機能自体は変わらない）

2. MINOR（マイナー）を上げる：1.0.1 → 1.1.0
    * 内容： 後方互換性がある機能追加
    * タイミング：
      * 新しいコマンドやオプションを追加した
      * 既存の機能を使いやすく拡張した（古い使い方もそのままできる場合）
    * 下の位（PATCH）は 0 にリセットする

3. MAJOR（メジャー）を上げる：1.1.0 → 2.0.0
    * 内容： 後方互換性がない破壊的な変更
    * タイミング：
      * 古い設定ファイルの書き方が使えなくなる変更
      * コマンド名を変えて、前の名前では動かなくなる場合
      * 依存しているライブラリの大幅な変更で、動作要件が変わる場合
    * 下の位（MINOR, PATCH）は 0 にリセットする

# 手順

1. バージョン更新の基準に従って`@CHANGELOG.md`を更新する

2. `@.claude-plugin/plugin.json`の`version`を更新する
